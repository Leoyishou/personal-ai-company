# 剪辑工程初始化

当用户想要开始一个新的视频剪辑项目时，初始化标准目录结构。

## 触发场景

- "新建剪辑工程"
- "初始化一个视频项目"
- "我要做一个新视频"
- "帮我建个剪辑目录"

## 工作流

### Step 1: 收集信息

通过 args 或 AskUserQuestion 获取：

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 项目名称 | 是 | 中括号包裹的项目名 | `【AI50】ipublish小红书自动发布` |
| 原始素材 | 否 | 素材文件路径（支持多个） | `/Users/liuyishou/tmp/video.mp4` |

### Step 2: 创建目录结构

项目根目录：`~/usr/pac/content-bu/output/视频剪辑/`

```bash
PROJECT_ROOT="$HOME/usr/pac/content-bu/output/视频剪辑"
PROJECT_NAME="【项目名称】"
PROJECT_DIR="$PROJECT_ROOT/$PROJECT_NAME"

mkdir -p "$PROJECT_DIR"/{01_原始素材,02_工程文件,03_导出成品,04_文案与设计,05_版权文件}
```

### Step 3: 生成模板文件

在项目根目录创建 `README.md`：

```markdown
# 项目信息

## 基本信息
- **项目名称**：{项目名称}
- **创建日期**：{YYYY-MM-DD}
- **发布平台**：
- **视频时长**：

## 素材来源
-

## 备注

```

在 `04_文案与设计/` 下创建 `脚本.md`：

```markdown
# 脚本

> **黄金三秒**：开头不寒暄、不铺垫，直接抛钩子（结果前置/悬念/反常识/视觉冲击）
> **好奇心闭环**：每 30-45s 一个小闭环，关闭前先打开下一个悬念
> **视觉节奏**：每 5s 至少 1 次视觉变化，消灭 >3s 无变化的死亡空间
> **字幕**：15-20 字/块，关键词变色/加大，与语音偏差 <200ms
> **信息密度**：每句话服务核心信息，删除一切冗余

## 选题


## 文案


## 分镜


```

### Step 4: 导入素材（可选）

如果用户提供了原始素材路径，复制到 `01_原始素材/`：

```bash
cp "素材路径" "$PROJECT_DIR/01_原始素材/"
```

### Step 5: 登记索引（可选）

追加到全局索引表 `$PROJECT_ROOT/全局索引表.csv`：

```
项目编号,项目名称,创建日期,状态,本地工程路径,移动硬盘素材路径,备注
```

项目编号格式：`YYYYMMDD_NNN`（当日序号）

## 目录说明

| 目录 | 用途 |
|------|------|
| `01_原始素材` | 原始录制/下载的视频、音频、图片 |
| `02_工程文件` | 剪辑工程文件（剪映/PR/Remotion 等） |
| `03_导出成品` | 最终导出的成品视频 |
| `04_文案与设计` | 脚本、文案、封面设计稿 |
| `05_版权文件` | 授权书、版权声明等 |

## 完成后输出

创建完成后，打印项目目录树确认：

```bash
ls -la "$PROJECT_DIR"/
```
