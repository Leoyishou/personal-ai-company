## 快速开始

### 示例 1: 首次使用 - 登录

**用户：**
```
请帮我登录小红书
```

**Claude 执行流程：**
1. 调用 `xiaohongshu_login` MCP 工具
2. 打开登录界面
3. 等待用户完成登录
4. 确认登录成功

**Claude 回复：**
```
我已经为你打开了小红书登录界面。请在浏览器中完成登录：
1. 扫码或输入账号密码
2. 完成验证
3. 登录成功后会自动保存登录状态

登录完成后，你就可以开始发布内容了！
```

## 图文发布

### 示例 2: 简单图文发布

**用户：**
```
帮我发一篇小红书笔记：
标题：我的日常穿搭
内容：今天分享一套简约风穿搭，白色衬衫搭配牛仔裤，舒适又时尚！

```

**Claude 执行流程：**
1. 检查登录状态
2. 验证参数：
   - 标题：9 个字符 ✓
   - 内容：32 个字符 ✓
   - 图片：检查文件存在 ✓
3. 调用 `xiaohongshu_publish_content` MCP 工具
4. 等待发布结果

**Claude 回复：**
```
✅ 发布成功！

你的小红书笔记已经发布：
- 标题：我的日常穿搭
- 内容长度：32 字符
- 图片数量：1 张

你可以在小红书 APP 中查看你的笔记。
```

```

### 示例 8: 视频文件检查失败

**用户：**
```
发布视频：
标题：旅行 Vlog
内容：我的旅行视频
视频：~/Videos/travel.mp4
```

**Claude 执行流程：**
1. 检查视频路径：不是绝对路径 ❌
2. 提示用户使用绝对路径

**Claude 回复：**
```
❌ 视频路径错误

视频文件必须使用绝对路径，不能使用 ~ 或相对路径。

你的路径：~/Videos/travel.mp4
正确示例：/Users/liuyishou/Videos/travel.mp4

请提供视频的完整绝对路径。
```

### 示例 9: 视频文件过大

**用户：**
```
发布视频：
标题：完整版教程
内容：...
视频：/path/to/large_video.mp4（假设 1.5GB）
```

**Claude 执行流程：**
1. 检查视频文件大小：1.5GB > 1GB ❌
2. 提示用户压缩视频

**Claude 回复：**
```
❌ 视频文件过大

你的视频文件大小为 1.5GB，超过了建议的 1GB 限制。

建议：
1. 使用视频压缩工具减小文件大小
2. 缩短视频时长
3. 降低视频分辨率（如 1080p 改为 720p）

压缩后请重新上传。
```

## 故障排查

### 示例 14: MCP 服务未启动

**用户：**
```
帮我发布小红书笔记
```

**Claude 检测到错误：**
```
❌ 无法连接到小红书 MCP 服务

可能的原因：
1. MCP 服务未启动
2. 服务地址配置错误
3. 端口被占用

解决方案：

1️⃣ 启动 MCP 服务：
```bash
cd /path/to/xiaohongshu-mcp
go run .
```

2️⃣ 确认服务运行：
访问 http://localhost:18060/mcp

3️⃣ 检查 Claude Code 配置：
```bash
claude mcp list
```

如果列表中没有 xiaohongshu-mcp，执行：
```bash
claude mcp add --transport http xiaohongshu-mcp http://localhost:18060/mcp
```

完成后请重试发布。
```

## 高级用法

### 示例 16: 使用环境变量管理路径

**用户：**
```bash
# 设置图片目录环境变量
export XHS_IMAGE_DIR="/Users/liuyishou/Pictures/xiaohongshu"

# 在 Claude Code 中
发布笔记，图片在 $XHS_IMAGE_DIR/today.jpg
```

**Claude 执行流程：**
1. 展开环境变量路径
2. 验证文件存在
3. 发布内容

### 示例 17: 定期发布（配合 cron）

**crontab 配置：**
```bash
# 每天早上 9 点发布早安内容
0 9 * * * /usr/local/bin/publish_daily.sh
```

**publish_daily.sh：**
```bash
#!/bin/bash
DATE=$(date +%Y-%m-%d)
echo "发布今日内容：早安分享 - $DATE" | claude
```
