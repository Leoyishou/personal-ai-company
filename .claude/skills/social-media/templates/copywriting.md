# 社媒文案撰写

为小红书、X、B站等平台撰写高质量文案。

## 触发条件

- "帮我写个文案"、"写一段小红书文案"
- "优化这段文案"、"改一下这个标题"
- 发布内容时自动调用（由 social-media-publish 触发）

---

## 一、风格规范（必须遵守）

### 1.1 禁止使用的元素

| 类型 | 禁止示例 | 原因 |
|------|----------|------|
| 数字圆圈 emoji | 1️⃣ 2️⃣ 3️⃣ | 廉价感 |
| 过度 emoji 堆砌 | 太好了！！！🎉🎉🎉 | 刷屏感 |
| 营销号腔调 | "震惊！""不转不是中国人" | low |

### 1.2 推荐的格式

**列表序号：**
- 纯数字：1. 2. 3.
- 中文数字：一、二、三
- 短横线：-

**强调方式：**
- 「直角引号」包裹关键词
- 适度使用 **加粗**（仅标题/关键词）

**语气：**
- 简洁直接，不啰嗦
- 有观点，不和稀泥
- 专业但不装逼

## 二、平台适配规则

### 2.1 小红书

| 字段 | 限制 | 建议 |
|------|------|------|
| 标题 | ≤20字 | 15字以内最佳 |
| 正文 | ≤1000字 | 300-500字易读 |
| 标签 | 数组格式 | 3-6个，不带# |

**标题公式：**
```
数字+结果：「3招搞定xxx」
身份+痛点：「程序员必看！xxx」
反常识：「原来xxx才是关键」
提问式：「为什么xxx？」
```

**正文结构：**
```
钩子（1句话抓注意力）
↓
核心内容（分点阐述）
↓
总结/金句收尾
```

### 2.2 X (Twitter)

| 字段 | 限制 | 建议 |
|------|------|------|
| 推文 | ≤280字符 | 200字符以内 |
| 图片 | ≤4张 | 1-2张最佳 |

**风格：**
- 一条推文一个观点
- 开门见山，不铺垫
- 可用 Thread 展开长内容

### 2.3 B站

| 字段 | 限制 | 建议 |
|------|------|------|
| 标题 | ≤80字 | 20-40字 |
| 简介 | 无硬性限制 | 100-300字 |
| 标签 | ≤12个 | 5-8个核心标签 |

## 三、工作流程

### Step 1: 理解需求

确认以下信息：
- 目标平台（小红书/X/B站）
- 内容主题
- 目标受众
- 是否有参考素材

### Step 2: 撰写初稿

根据平台规则和风格规范撰写：
- 标题（多备选 2-3 个）
- 正文
- 标签建议

### Step 3: 输出格式

```yaml
平台: 小红书
标题: xxx
正文: |
  xxx
标签:
  - 标签1
  - 标签2
```

### Step 4: 用户确认

展示文案供用户审核，根据反馈修改。

## 四、文案模板库

### 4.1 知识科普类

```
标题：「核心概念」+ 疑问/反常识

正文：
很多人以为 xxx，其实 xxx。

核心原因有三个：

1. xxx
   - 细节解释

2. xxx
   - 细节解释

3. xxx
   - 细节解释

结论：xxx

想了解更多？xxx
```

### 4.2 工具推荐类

```
标题：数字 + 场景 + 效果

正文：
作为 [身份]，这几个工具我天天在用：

1. [工具名] - 一句话说明
2. [工具名] - 一句话说明
3. [工具名] - 一句话说明

个人最推荐 [xxx]，因为 xxx。

你们还有什么好用的？评论区交流
```

### 4.3 经验分享类

```
标题：身份 + 时间 + 结果

正文：
[背景铺垫，1-2句]

踩过的坑：
- xxx
- xxx

现在的做法：
- xxx
- xxx

最大的收获是 xxx。
```
