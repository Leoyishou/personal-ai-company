# PMO 项目归类规则 - 投资事业部

## 一、核心概念

### 1.1 Session 与 Issue 的关系

```
1 个本地 Session = 1 或 n 个 Linear Issue
```

- 每个 Issue 的 description 必须包含 `sessionId: <session_id>`
- 交易和调研是两种不同类型的 Issue

### 1.2 事业部信息

| 字段 | 值 |
|------|-----|
| Team ID | TODO: 创建后更新 |
| Key | I |
| 说明 | 投资决策与执行 |

## 二、Project = 投资类型

投资事业部的 Project 按资产类别划分：
- `美股` - 美股交易与调研
- `港股` - 港股交易与调研
- `加密货币` - 数字资产
- `其他资产` - 债券、期权等

## 三、Labels

### 3.1 活动类型

| Label | 说明 |
|-------|------|
| 交易 | 实际买入/卖出操作 |
| 调研 | 深度研究分析 |
| 复盘 | 交易复盘、策略检验 |
| 观察 | 加入观察列表 |

### 3.2 决策依据（可选）

| Label | 说明 |
|-------|------|
| 基本面 | 基于财务/业务分析 |
| 技术面 | 基于图表/指标 |
| 情绪面 | 基于市场情绪 |
| 事件驱动 | 基于特定事件 |

## 四、Issue 创建规范

### 4.1 交易 Issue

**标题格式**：`【MMdd-HH】交易：{股票} {方向}`

**Description 模板（轻量摘要）**：

```markdown
sessionId: <session_id>

## 摘要
<标的> <方向> <数量>股 @ $<价格>

## 仓位变化
- 交易前: xx%
- 交易后: xx%
```

**Document: 交易决策**（通过 `doc-create --issue I-xx` 创建）：

```markdown
## 交易信息
- **标的**: {股票代码} {股票名}
- **方向**: 买入/卖出
- **数量**: xxx 股
- **价格**: $xxx
- **时间**: {ISO时间}

## 决策依据
<为什么做这笔交易>

## 后续计划
- 止盈: $xxx
- 止损: $xxx
```

### 4.2 调研 Issue

**标题格式**：`【MMdd-HH】调研：{主题}`

**Description 模板（轻量摘要）**：

```markdown
sessionId: <session_id>

## 摘要
<一句话核心结论>

## 投资建议
- 行动: 买入/持有/观望/卖出
```

**Document: 调研报告**（通过 `doc-create --issue I-xx` 创建）：

```markdown
## 调研主题
<研究什么>

## 核心结论
1. ...
2. ...
3. ...

## 数据支撑
<关键数据和来源>

## 投资建议
- 行动: 买入/持有/观望/卖出
- 理由: ...
- 风险: ...

## 参考链接
- ...
```

## 五、触发与归类

### 5.1 触发时机

| 事件 | Hook | 说明 |
|------|------|------|
| `futu-trades` skill | pmo-report-trade.js | 检测实际交易 |
| `research` skill (在投资目录) | pmo-report-research.js | 检测深度调研 |

### 5.2 交易检测规则

只有包含以下关键词的输出才算实际交易：
- "买入"、"卖出"、"成交"
- "filled"、"executed"

查询类操作（查看持仓、查看行情）不创建 Issue。

### 5.3 调研检测规则

只在 `investment-bu` 目录下触发 `research` skill 时才上报。
其他目录的调研不算投资事业部活动。

## 六、关联逻辑

### 6.1 交易关联调研

如果同一 Session 中：
1. 先做了调研
2. 后执行了交易

应将交易 Issue 与调研 Issue 关联：
```graphql
mutation {
  issueRelationCreate(input: {
    issueId: "交易-issue-id",
    relatedIssueId: "调研-issue-id",
    type: "relates_to"
  })
}
```

### 6.2 交易复盘

交易后 7 天自动创建复盘提醒：
- 检查盈亏情况
- 评估决策质量
- 更新策略

## 七、注意事项

- 交易记录是敏感信息，确保只在本地 Linear 可见
- 调研报告可能包含第三方版权内容，注意引用
- 定期复盘，沉淀有效策略
- 量化指标：胜率、盈亏比、年化收益

## 八、Session 总结模板

Session 结束时，通过 `doc-create --issue I-xx` 创建 Document（而非追加到 description）：

**Document 标题**：`Session 总结: <投资主题>`

```markdown
## 研究/操作概要
- 关注标的: ...
- 主要结论: ...
- 执行动作: ...

## 信息来源
- 公开信息: ...
- 第三方分析: ...
- 自有模型: ...

## 风险提示
- 已识别风险: ...
- 对冲措施: ...

## 下一步
- 待观察: ...
- 待执行: ...
```
